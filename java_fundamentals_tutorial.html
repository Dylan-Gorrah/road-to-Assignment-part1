Section 01

Variables & Data Types

A variable is just a named box that holds a piece of information. You give the box a name, and you can put something in it and read it later.

But before you create a box in Java, you have to tell it what kind of thing it's going to hold. That's the data type. Think of it like labelling the box — "this box is for words", "this box is for numbers", "this box is for yes/no answers".

The four types you'll use most

String
Text — any word, sentence, or mix of characters. Always goes in double quotes.
int
A whole number — no decimals. 1, 42, -10, 0.
boolean
Only two possible values: true or false. Yes or no. On or off.
char
A single character — one letter, digit, or symbol. Goes in single quotes: 'A', '3', '_'.
Variables.java
public class Variables {
    public static void main(String[] args) {

        // String — holds text. Always in double quotes.
        String firstName = "Kyle";
        String lastName  = "Smith";

        // int — whole numbers only. No quotes.
        int age  = 21;
        int score = 95;

        // boolean — only true or false. No quotes.
        boolean isLoggedIn = false;
        boolean isAdmin    = true;

        // char — one single character. Always in single quotes.
        char initial  = 'K';
        char underscore = '_';

        // Printing them out to the console
        System.out.println("Name: " + firstName + " " + lastName);
        System.out.println("Age: " + age);
        System.out.println("Logged in: " + isLoggedIn);
        System.out.println("Initial: " + initial);
    }
}
Output
Name: Kyle Smith Age: 21 Logged in: false Initial: K
!

Notice the pattern: type → name → = → value → ;
String firstName = "Kyle";
Every variable declaration in Java follows that same structure. The semicolon at the end is not optional — Java will refuse to run without it.

!

Why does this matter for the login app? Because every piece of data we handle — the username, password, first name, last name, cell number — is a String. And every check we run returns a boolean. These are the two types you'll use most.

Section 02

If / Else — Making Decisions

An if/else statement lets your code make a decision. "If this condition is true — do this. Otherwise (else) — do that." It's the most used thing in programming.

Read it like a sentence: "If the password is long enough — accept it. Otherwise — reject it."

IfElse.java
public class IfElse {
    public static void main(String[] args) {

        int age = 17;

        // The condition goes inside the ( ) brackets.
        // If it's true, the first block runs. If not, the else block runs.
        if (age >= 18) {
            System.out.println("You are old enough to vote.");
        } else {
            System.out.println("You are too young to vote.");
        }

        // You can also chain them with else if
        // for more than two outcomes:
        int score = 74;

        if (score >= 75) {
            System.out.println("Distinction!");
        } else if (score >= 50) {
            System.out.println("Pass.");
        } else {
            System.out.println("Did not pass.");
        }
    }
}
Output
You are too young to vote. Pass.
!

The curly braces { } group everything that belongs to the if or else. Everything inside the braces only runs when that condition is true. Don't leave them out — it leads to subtle bugs that are a nightmare to find.

Section 03

Boolean Logic — &&, ||, and !

Sometimes one condition isn't enough. What if you need both things to be true? Or at least one? That's where boolean operators come in.

&& (AND)
Both sides must be true for the whole thing to be true.
|| (OR)
At least one side must be true. If either is true, the whole thing is true.
! (NOT)
Flips a boolean. true becomes false. false becomes true.
BooleanLogic.java
public class BooleanLogic {
    public static void main(String[] args) {

        boolean hasUnderscore = true;
        boolean shortEnough   = false; // username is too long

        // AND — both must be true
        boolean validUsername = hasUnderscore && shortEnough;
        System.out.println("Valid username: " + validUsername); // false

        // OR — at least one must be true
        boolean eitherPasses = hasUnderscore || shortEnough;
        System.out.println("Either passes: " + eitherPasses); // true

        // NOT — flip it
        boolean isInvalid = !validUsername;
        System.out.println("Is invalid: " + isInvalid); // true

        // Real example from the login app:
        // The username check returns true only if BOTH conditions pass.
        if (hasUnderscore && shortEnough) {
            System.out.println("Username successfully captured.");
        } else {
            System.out.println("Username is not correctly formatted.");
        }
    }
}
Output
Valid username: false Either passes: true Is invalid: true Username is not correctly formatted.
!

In the login app, every single validation method is basically just boolean logic. checkUserName() does return hasUnderscore && shortEnough. That's it. Once you get comfortable with &&, that whole method just clicks.

Section 04

Methods — Reusable Blocks of Code

A method is a named block of code that does one specific job. You write it once, and then you can call it as many times as you want. Think of it like a vending machine button — you press B3, it always dispenses the same thing.

Methods can optionally take in some information (parameters), and they can optionally send something back when they're done (return value).

The anatomy of a method

public
Who can use this method. public = anyone. (More on this later.)
boolean / String / void
What type of thing it sends back. void = nothing, it just runs.
methodName()
The name you'll use to call it. Brackets hold the parameters.
return ...
Sends a value back to whoever called the method, then stops running.
Methods.java
public class Methods {

    // A method that takes a name and returns a greeting.
    // Return type: String  |  Parameter: String name
    public static String greet(String name) {
        return "Hello, " + name + "!";
    }

    // A method that checks if a number is positive.
    // Return type: boolean  |  Parameter: int number
    public static boolean isPositive(int number) {
        return number > 0;
    }

    // A method that just prints something. No return value.
    // Return type: void  |  No parameters
    public static void printDivider() {
        System.out.println("──────────────────────");
    }

    public static void main(String[] args) {

        // Calling the methods and using their return values
        String message = greet("Amahle");
        System.out.println(message);        // Hello, Amahle!

        System.out.println(greet("Kyle"));  // Hello, Kyle!

        boolean result = isPositive(42);
        System.out.println(result);          // true

        System.out.println(isPositive(-5)); // false

        printDivider();                      // ──────────────────────
    }
}
Output
Hello, Amahle! Hello, Kyle! true false ──────────────────────
!

See how we call greet("Amahle") and greet("Kyle")? Same method, different input, different output. That's the whole point of methods — write the logic once, use it everywhere. In the login app, we write checkUserName() once and the app calls it every time a user registers.

Section 05

Classes & Objects

A class is a blueprint. An object is the actual thing built from that blueprint. Think of a class as the architectural plans for a house. The object is the actual house you build from those plans. You can build many houses from the same plans — each one is its own separate object with its own data.

An object holds two things: data (variables inside it) and behaviour (methods it can perform). This is the core idea of Object-Oriented Programming — OOP.

Dog.java + Main.java
// ── The Blueprint (class) ──────────────────────────────────
public class Dog {

    // Data — every Dog object will have these
    String name;
    String breed;
    int age;

    // Behaviour — things a Dog object can do
    public void bark() {
        System.out.println(name + " says: Woof!");
    }

    public void introduce() {
        System.out.println("Hi, I'm " + name + ", a " + breed + ".");
    }
}

// ── Using It (creating objects from the blueprint) ─────────
public class Main {
    public static void main(String[] args) {

        // new Dog() builds an actual object from the Dog class.
        // dog1 and dog2 are completely separate — changing one
        // doesn't affect the other.
        Dog dog1 = new Dog();
        dog1.name  = "Rex";
        dog1.breed = "German Shepherd";
        dog1.age   = 3;

        Dog dog2 = new Dog();
        dog2.name  = "Buddy";
        dog2.breed = "Labrador";
        dog2.age   = 5;

        dog1.bark();
        dog2.bark();
        dog1.introduce();
    }
}
Output
Rex says: Woof! Buddy says: Woof! Hi, I'm Rex, a German Shepherd.
!

In the login app, Login is a class. When Main.java does Login login = new Login();, it builds one Login object. That one object holds the saved username, password, and name. We pass it between frames so they all share the exact same data — changing the name inside RegisterFrame is visible in LoginFrame and WelcomeFrame, because it's the same object.

Section 06

Constructors

A constructor is a special method that runs automatically the moment you create a new object. You write it inside the class, and Java calls it whenever someone does new ClassName(...).

It looks like a method but it has no return type, and its name must exactly match the class name. Its job is usually to set up the object's initial data.

Student.java
public class Student {

    String name;
    int studentNumber;
    String course;

    // This IS the constructor.
    // It has the same name as the class, no return type.
    // It runs the moment you write: new Student("Kyle", 12345, "IT")
    public Student(String name, int studentNumber, String course) {
        // "this.name" = the variable stored in this object
        // "name"      = the value that was passed in as a parameter
        this.name          = name;
        this.studentNumber = studentNumber;
        this.course        = course;
    }

    public void printInfo() {
        System.out.println("Name:   " + name);
        System.out.println("Number: " + studentNumber);
        System.out.println("Course: " + course);
    }

    public static void main(String[] args) {

        // The constructor runs right here — the three values
        // go straight into name, studentNumber, and course.
        Student s1 = new Student("Kyle Smith", 12345, "IT");
        Student s2 = new Student("Amahle Dube", 67890, "Computer Science");

        s1.printInfo();
        System.out.println("---");
        s2.printInfo();
    }
}
Output
Name: Kyle Smith Number: 12345 Course: IT --- Name: Amahle Dube Number: 67890 Course: Computer Science
!

In the login app, every frame class has a constructor that takes a Login object as a parameter — public RegisterFrame(Login login). That's why in Main.java we do new RegisterFrame(login). We're calling the constructor and passing our Login object in.

Section 07

Private vs Public — Controlling Access

When you put a variable or method inside a class, you can control who's allowed to see it or use it. This is called access control, and it's one of the most important ideas in OOP.

public
Anyone, anywhere can read or change this. No restrictions.
private
Only code inside this same class can touch it. Completely hidden from outside.

So why would you want something to be private? Protection. If your Login class stores a password, you don't want other classes randomly changing it. Private keeps it locked away — you can only change it through the methods you've approved.

Getters — the approved door

A getter is a small public method whose only job is to return the value of a private variable. It's the "approved door" through which the outside world can read a private value without being able to change it directly.

BankAccount.java
public class BankAccount {

    // private — nobody outside this class can change these directly.
    // Try doing account.balance = 1000000 from another class — it won't compile.
    private String owner;
    private double balance;

    public BankAccount(String owner, double startingBalance) {
        this.owner   = owner;
        this.balance = startingBalance;
    }

    // Getter — the only approved way to read the balance.
    // Notice it's public so others can call it, but it returns
    // a copy of the value — they can't write to it.
    public double getBalance() {
        return balance;
    }

    public String getOwner() {
        return owner;
    }

    // The ONLY approved way to add money — with a safety check.
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited R" + amount);
        } else {
            System.out.println("Invalid deposit amount.");
        }
    }

    public static void main(String[] args) {
        BankAccount acc = new BankAccount("Kyle", 500.00);

        // acc.balance = 9999;  ← this would cause a compile error. Good.

        acc.deposit(250.00);
        System.out.println("Balance: R" + acc.getBalance());
    }
}
Output
Deposited R250.0 Balance: R750.0
!

In the login app, we use getFirstName() and getLastName() in Login.java for exactly this reason. The names are stored as private variables. The WelcomeFrame can read them via the getter, but it can't accidentally overwrite them.

Section 08

String Manipulation

A String is just text. But Java gives you a bunch of built-in tools to work with that text — counting characters, checking what's inside it, comparing it to another String, and more. These are called String methods.

You use them with dot notation: myString.methodName().

.length()
Returns how many characters are in the String.
.contains("x")
Returns true if the String includes the given text somewhere inside it.
.equals("x")
Returns true if the String's text is exactly the same as "x". Use this instead of ==.
.toCharArray()
Splits the String into an array of individual characters so you can loop through them.
.matches("pattern")
Tests whether the String matches a regex pattern. Returns true or false.
.startsWith("x")
Returns true if the String begins with the given text.
.toUpperCase()
Returns a copy of the String with all letters converted to capitals.
Strings.java
public class Strings {
    public static void main(String[] args) {

        String username = "kyl_1";
        String password = "MyPass@99";
        String cell     = "+27838968976";

        // .length() — how many characters?
        System.out.println(username.length());     // 5

        // .contains() — does it include this text?
        System.out.println(username.contains("_")); // true
        System.out.println(username.contains("@")); // false

        // .equals() — same text?  ALWAYS use this, never ==
        String entered = "kyl_1";
        System.out.println(username.equals(entered)); // true

        // .startsWith() — does it start with +27?
        System.out.println(cell.startsWith("+27"));    // true

        // .toUpperCase() — turn it all caps
        System.out.println(username.toUpperCase());    // KYL_1

        // .matches() with a simple regex pattern
        // \\+27[0-9]{9} means: starts with +27, then exactly 9 digits
        System.out.println(cell.matches("\\+27[0-9]{9}")); // true

        // .toCharArray() + looping through each character
        // This is how we check the password for capital letters etc.
        for (char c : password.toCharArray()) {
            if (Character.isUpperCase(c)) {
                System.out.println("Found a capital: " + c);
            }
        }
    }
}
Output
5 true false true true KYL_1 true Found a capital: M Found a capital: P
!

Why not ==? When you write username == entered, Java doesn't compare the text — it checks whether the two variables point to the exact same chunk of memory. That can give you false even when the words are identical. .equals() always compares the actual text. This mistake has caught out basically every Java beginner at least once.

Section 09

For Loops

A loop runs a block of code repeatedly — either a set number of times, or once for each item in a collection. Instead of writing the same line 10 times, you write it once inside a loop and let it run.

The standard for loop

Three parts inside the brackets, separated by semicolons: start → condition → step. It keeps running as long as the condition is true.

Loops.java
public class Loops {
    public static void main(String[] args) {

        // Standard for loop — runs 5 times (i goes 0, 1, 2, 3, 4)
        //   int i = 0   → start with i at 0
        //   i < 5       → keep going while i is less than 5
        //   i++         → after each loop, add 1 to i
        for (int i = 0; i < 5; i++) {
            System.out.println("Loop number: " + i);
        }

        System.out.println("---");

        // For-each loop — go through every character in a String.
        // "for each char c in the character array of password..."
        // This is exactly what the password complexity check uses.
        String password = "MyPass@99";
        boolean hasUpper   = false;
        boolean hasDigit   = false;
        boolean hasSpecial = false;

        for (char c : password.toCharArray()) {
            if (Character.isUpperCase(c))       hasUpper   = true;
            if (Character.isDigit(c))           hasDigit   = true;
            if (!Character.isLetterOrDigit(c)) hasSpecial = true;
        }

        System.out.println("Has capital:  " + hasUpper);
        System.out.println("Has digit:    " + hasDigit);
        System.out.println("Has special:  " + hasSpecial);
    }
}
Output
Loop number: 0 Loop number: 1 Loop number: 2 Loop number: 3 Loop number: 4 --- Has capital: true Has digit: true Has special: true
!

The for-each loop for (char c : password.toCharArray()) reads as: "for each character c in the array of characters from password." Each time through the loop, c is the next character. So for "MyPass@99", c goes: M → y → P → a → s → s → @ → 9 → 9.

countDigits method
public static int countDigits(String text) {

    int count = 0; // start at zero

    for (char c : text.toCharArray()) {
        if (Character.isDigit(c)) {
            count++; // count++ means add 1 to count
        }
    }

    return count;
}

// Test it:
System.out.println(countDigits("abc123"));     // 3
System.out.println(countDigits("hello"));      // 0
System.out.println(countDigits("p@ssW0rd99")); // 3
!

count++ is shorthand for count = count + 1. Start at 0, loop through every character, add 1 each time you find a digit, then return the total. This exact same counter pattern is used in more complex form inside checkPasswordComplexity().
